白嫖甲骨文云：永久免费 ARM 服务器申请攻略

详细记录了申请 Oracle Cloud Always Free Tier 的全过程，以及如何避免账号被封和防回收的实用技巧。

为什么是甲骨文云

  • 真正的永久免费：只要账号正常，资源不清零、不要求升级付费。
  • ARM 实例 强悍：最高可免费开通 4 核 ARM / 24 GB 内存 的虚拟机。
  • 带宽慷慨：出站带宽约 480 Mbps～1 Gbps，对个人项目绰绰有余。
  • 两个区域可选：主区域 + 另外一个免费区域（如首尔、东京、春川等）。
  • 相比 AWS/GCP 免费套餐：不要求一年后主动停机，更省心。

⚠️ 但众所周知：注册难度高、封号无理由。本文会聚焦技巧与避坑。

准备工作（决定成败）

必备条件

项目 要求与建议
信用卡 Visa / Mastercard（支持银联部分卡，但建议非银联外币卡）
邮箱 建议 Gmail / Outlook / 企业邮，不要用临时邮箱
手机号 真实且能接收短信（国内手机号可行）
网络环境 从始至终保持 住宅 IP + 不挂代理
浏览器 无痕模式 / 清过缓存的 Chrome 或 Firefox
地址信息 与信用卡账单地址完全一致（拼音即可）

高风险行为（做了基本被拒）

  • 使用机房 IP（阿里云/腾讯云/VPN 节点）注册
  • 信用卡曾被其他 Oracle Cloud 账号绑定过
  • 短时间内反复提交申请
  • 名字用明显假名（如 Test User）

第一步：选择区域与注册入口

Oracle 的免费区域是有 “热门程度” 区分的：

区域 ARM 资源可用性 速度（国内） 推荐度
春川 (Chuncheon) ⭐⭐ 经常无货 优秀 极高
首尔 (Seoul) ⭐ 极难开 优秀 中
东京 (Tokyo) ⭐⭐ 偶尔有货 较好 中高
大阪 (Osaka) ⭐⭐ 中等 较好 中
圣何塞 (San Jose) ⭐⭐⭐ 较充足 一般 中
法兰克福 ⭐⭐⭐ 充足 较差 低

策略：注册时填的主区域要选 ARM 容易开的（如春川、圣何塞）。
入口：直接访问 https://signup.cloud.oracle.com

第二步：注册流程逐项拆解

2.1 填写身份与联系信息

  • First Name / Last Name：用拼音，与信用卡姓名一致（如 San Zhang）
  • Email：一定要能长期使用
  • Country / Region：选你信用卡所在国家（中国选 China）
  • Home Region：关键决策，选你想要的免费区域（不可轻易更改）

2.2 验证手机号

  • 选择 China (+86)
  • 输入手机号，接收短信验证码
  • 不要使用虚拟号段（170/171 等）

2.3 填写地址与信用卡

  • 地址写拼音即可：No.XX, XX Road, XX District
  • 信用卡信息：持卡人姓名用拼音
  • 扣费说明：会发起 1 美元 预授权（验证卡有效性），很快退回，不会真实扣费

如果这一步报错 Transaction failed 或 Credit card not approved，大概率是卡的问题。建议换一张外币卡。

2.4 等待邮件

提交后会收到：

  • 即时邮件：We are processing your request
  • 几分钟到几个小时 后收到最终邮件：
    • ✅ Your Cloud Account is Ready → 成功
    • ❌ Unable to process your registration → 被拒（换卡/换环境重试）

第三步：登录与创建 ARM 虚拟机（最容易被坑）

成功注册后，登录控制台：
https://cloud.oracle.com

3.1 免费资源速查

在 Compute → Instances 中，Always Free 配额是：

  • ARM (Ampere A1)：最多 4 核 / 24 GB 内存（可拆分成多台，总和不超过）
  • AMD (VM.Standard.E2.1.Micro)：1 核 / 1 GB 内存（老一代免费机）
  • 总计 200 GB 块存储（可挂载卷）
  • 每月 10 TB 出站流量

建议：直接创建 4 核 + 24 GB 内存的 ARM 实例（性能最强）。

3.2 创建步骤

  1. 点击 Create instance
  2. 名称随意（如 arm-free）
  3. Placement：保留 Availability domain
  4. Image：选择 Canonical Ubuntu 22.04（最稳妥）或 Oracle Linux
  5. Shape → Change shape：
    • 选择 Specialty and legacy → Ampere
    • 选择 VM.Standard.A1.Flex
    • 配置 OCPUs: 4，Memory: 24 GB
  6. Networking：保留默认虚拟云网络
  7. Add SSH keys：
    • 生成 SSH 密钥对（本地 ssh-keygen），粘贴 公钥 内容
  8. Boot volume：默认 100 GB（免费额度够用）
  9. 点击 Create

3.3 常见失败及解决方法

错误提示 原因 解决方法
Out of capacity for shape VM.Standard.A1.Flex 该区域 ARM 资源缺货 等待（凌晨/月初尝试）或换区域
Service limit exceeded 配额已达 检查是否已有其他免费实例
Internal error 临时故障 换个浏览器或 30 分钟后重试

抢货技巧：

  • 使用 OCI CLI 或 脚本 自动重试（本文末尾提供思路）
  • 关注 Telegram 频道：@oraclefreers（实时通报有货区域）

第四步：防止账号被封的核心技巧

甲骨文封号特点：不讲武德。很多正常用户也被封。以下是经过验证的存活策略。

✅ 一定要做的

  1. 开启 MFA（多因素认证）

    • 控制台右上角 → 用户设置 → Security → Multi-factor authentication
    • 用 Google Authenticator / Microsoft Authenticator 绑定

  2. 升级到付费账户（免费资源仍免费）
    这不叫“付费”，而是身份验证。升级后封号概率大幅降低。

    • 导航到 Billing → Upgrade Subscription
    • 绑定同一张信用卡（会验证但不会扣费）
    • 升级后保持免费资源不变

  3. 保持活跃

    • 每月至少登录控制台 2-3 次
    • 避免实例完全闲置（跑个小服务）

  4. 不要挖矿、发垃圾邮件、BT 下载
    这是绝对红线。

❌ 绝对不要做的

  • 将实例用于商业盈利（跑企业 API）
  • 尝试暴力破解他人系统
  • 创建大量小规格实例（被判定为滥用）
  • 用脚本频繁创建/删除实例

账号被回收的“保护期”说法

  • 甲骨文对 90 天以上无活跃资源 的免费账号有回收可能
  • 防回收方案：在实例上部署一个简单的 HTTP 服务（python -m http.server）并配置 cron 每天调用一次外部健康检查（如 uptimerobot）
